# CHANGELOG

### 名词解释

在更新日志中会包含以下几种名词：

1.  “**新增**”：

    -   追加了新功能；

    -   一般不会产生破坏性更新。

2.  “**变更**”：

    -   可能重新实现了已有的功能，通常会带来使用方式上的变化；

    -   可能升、降级了第三方依赖库的版本。

    -   一般会产生破坏性更新。

3.  “**修复**”：

    -   修复了已知的一些问题；

    -   不会产生破坏性更新。
